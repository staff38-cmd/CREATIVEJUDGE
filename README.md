# CREATIVEJUDGE

広告クリエイティブの薬機法・景品表示法・媒体審査対応を、AIが自動チェックする社内プラットフォームです。
「このまま出したら審査に落ちるか・CLにNGを出されるか」を事前に見抜くことを目的としています。

---

## 主な機能

### AIチェック
- **マルチコンテンツ対応** — 画像・動画・テキスト原稿・LP・URLに対応
- **一括テキストチェック** — 1行1件でTDやキャッチコピーを複数まとめてチェック
- **実務観点での判定** — 理論上の法令解釈ではなく、媒体審査・CLチェックで実際に引っかかるかの視点で判定
- **過去NG事例との直接照合** — 案件ごとのNG事例と文字列マッチングで確実に検出（AIの見落とし防止）
- **媒体別レギュレーション** — Meta / Google / ByteDance / LINE / SmartNews / YDA の媒体固有ルールを反映

### 案件・ナレッジ管理
- **案件管理** — クライアント単位でNG事例・許容表現・企業レギュレーションを紐付け
- **NG事例同期** — GoogleスプレッドシートのNG集をAPIで自動インポート
- **許容表現登録** — 「これはOKと確認済み」の表現を登録してAIの誤検知を防止

### チェックリスト
- **媒体×CR種別ごとのチェックリスト** — バナー・動画・TD・入稿の必須確認項目を記録
- **二重チェック記名** — 担当者・レビュアーの記名と承認状態を管理
- **提出ブロック** — NG項目が残っている場合に警告表示

### ダッシュボード
- 月別チェック・NG推移グラフ
- 規制カテゴリ別NG件数ランキング
- 案件別チェック・NG件数
- よく出るNG表現 TOP5
- 媒体別チェックリスト件数

## チェック観点

| 観点 | 内容 |
|------|------|
| 薬機法 | 医薬品的断言、効果効能の断定表現 |
| 景品表示法 | 根拠なきNo.1・最上級、受注速報・閲覧人数の虚偽表示 |
| ステマ規制 | 「愛用者」「購入者」等のステマ口コミ表現 |
| 媒体審査 | 媒体ごとの固有NGルール（Meta・Google・ByteDance等） |
| 企業レギュレーション | クライアント固有の禁止事項（案件ごとに設定） |
| 過去NG事例 | 実際にNGになった表現との直接照合 |

## ローカル開発

```bash
npm install
cp .env.example .env.local
# .env.local に必要な環境変数を設定
npm run dev
```

[http://localhost:3000](http://localhost:3000) を開いてください。

### 必要な環境変数

```
GEMINI_API_KEY=           # Google Gemini API キー
JWT_SECRET=               # セッション署名用シークレット
GOOGLE_CREDENTIALS_PATH=  # Google Sheets API 認証情報（JSONファイルパス）
```

## デプロイ

mainブランチへのpushで GitHub Actions が自動実行します。

1. GitHub Actions でビルド（Ubuntu / メモリ制限なし）
2. ビルド済み `.next/` をサーバーに rsync 転送
3. サーバーで `pm2 restart`（数秒のダウンタイムのみ）
